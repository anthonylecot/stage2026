Agenda Ateliers Checklist

HTML + CSS + GitHub Pages

Durée : 3 sessions (2h + 2h + 2h)
Matériel : Ordinateurs, VS Code, comptes GitHub
Groupes : Groupe A (S1 lun–mar), Groupe B (S2 ven tutorat + S3 lun)

Objectif pédagogique

À la fin de cet atelier, les stagiaires auront publié un site HTML personnel sur GitHub Pages qui leur sert de portfolio pour toute la semaine — une nouvelle page ajoutée après chaque atelier.

Session 1 — Structure de base (2h)

Mise en place de VS Code

  1. Ouvrir VS Code
  2. Créer un dossier mon-site sur le bureau
  3. Dans VS Code : File → Open Folder → sélectionner mon-site
  4. Dans l’explorateur à gauche : clic droit → New File → nommer index.html

VS Code en 30 secondes : explorateur de fichiers à gauche, éditeur au centre, terminal en bas (Ctrl+ù ou Terminal → New Terminal). Pour ouvrir le fichier dans le navigateur : clic droit sur index.html → Open with Live Server (si l’extension est installée), ou glisser le fichier dans Chrome.

Structure de base d’une page HTML

Coller ce squelette dans index.html :

<!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="UTF-8">
  <title>Mon site</title>
</head>
<body>

  <h1>Bonjour, je m'appelle Prénom</h1>
  <p>Je suis stagiaire chez SAP en juin 2026.</p>

</body>
</html>

Explication ligne par ligne :

Balise Rôle
<!DOCTYPE html> Dit au navigateur que c’est du HTML5
<html lang="fr"> Début du document, langue française
<head> Informations sur la page (non visibles)
<meta charset="UTF-8"> Permet les accents et caractères spéciaux
<title> Texte affiché dans l’onglet du navigateur
<body> Tout ce qui s’affiche dans la page
<h1> Titre principal (il n’y en a qu’un par page)
<p> Paragraphe

Ouvrir dans Chrome → vérifier que ça s’affiche.

Les balises essentielles

Faire tester chacune en l’ajoutant dans le <body> :

<!-- Titres -->
<h1>Titre principal</h1>
<h2>Sous-titre</h2>
<h3>Sous-sous-titre</h3>

<!-- Texte -->
<p>Un paragraphe de texte.</p>
<strong>Texte en gras</strong>
<em>Texte en italique</em>

<!-- Listes -->
<ul>
  <li>Élément 1</li>
  <li>Élément 2</li>
</ul>

<!-- Lien -->
<a href="https://sap.com">Visiter SAP</a>

<!-- Image -->
<img src="photo.jpg" alt="Description de l'image">

<!-- Saut de ligne -->
<br>

<!-- Ligne horizontale -->
<hr>

Liens entre pages

Quand le site a plusieurs pages, il faut naviguer entre elles.

Depuis index.html vers une autre page :

<a href="agenda.html">Mon agenda</a>

Lien retour dans agenda.html :

<a href="index.html">← Retour à l'accueil</a>

Lien vers une section dans la même page (ancre) :

<!-- La cible -->
<h2 id="semaine1">Semaine 1</h2>

<!-- Le lien -->
<a href="#semaine1">Voir semaine 1</a>

Barre de navigation commune à toutes les pages :

<nav>
  <a href="index.html">Accueil</a>
  <a href="agenda.html">Agenda</a>
  <a href="scratch.html">Scratch</a>
</nav>

Copier-coller la même <nav> dans chaque fichier HTML — navigation identique sur tout le site.

Tableaux

Structure de base :

<table>
  <tr>
    <th>Jour</th>
    <th>Matin</th>
    <th>Après-midi</th>
  </tr>
  <tr>
    <td>Lundi</td>
    <td>HTML</td>
    <td>HTML CSS</td>
  </tr>
  <tr>
    <td>Mardi</td>
    <td>CSS + interview</td>
    <td>OpenCode</td>
  </tr>
</table>

CSS pour le rendre lisible :

table {
  border-collapse: collapse;
  width: 100%;
}

th, td {
  border: 1px solid #cccccc;
  padding: 8px 12px;
  text-align: left;
}

th {
  background-color: #0070ad;
  color: white;
}

L’agenda du stage est l’exercice naturel : jours en lignes, matin/après-midi en colonnes.

Chaque stagiaire crée sa page de présentation dans index.html :

  • Un <h1> avec son prénom
  • Un <p> de présentation (2–3 phrases)
  • Une liste <ul> de 3 centres d’intérêt
  • Un lien vers un site qu’il aime

Pas de CSS encore — l’objectif c’est que ça marche, pas que ce soit beau. Le moche est temporaire.

<a href="agenda.html">Mon agenda</a>

Session 2 — CSS (2h)

Créer le fichier CSS

Dans VS Code, créer style.css dans le même dossier que index.html.

Lier le CSS dans le <head> de chaque page HTML :

<link rel="stylesheet" href="style.css">

Premiers styles

/* Couleur de fond et texte général */
body {
  background-color: #f5f5f5;
  color: #333333;
  font-family: Arial, sans-serif;
  max-width: 800px;
  margin: 0 auto;
  padding: 20px;
}

/* Titre principal */
h1 {
  color: #0070ad;
}

/* Liens */
a {
  color: #0070ad;
}

/* Paragraphes */
p {
  line-height: 1.6;
}

Recharger la page dans Chrome — voir la différence.

Propriétés CSS à explorer :

Propriété Exemple Effet
color color: red; Couleur du texte
background-color background-color: #f0f0f0; Couleur de fond
font-size font-size: 18px; Taille du texte
font-family font-family: Georgia; Police
margin margin: 20px; Espace extérieur
padding padding: 10px; Espace intérieur
border border: 1px solid black; Bordure
text-align text-align: center; Alignement

Classes CSS

Pour styliser certains éléments différemment :

Dans le HTML :

<p class="important">Ce texte est important.</p>

Dans le CSS :

.important {
  background-color: #fff8e1;
  border-left: 4px solid #ffc107;
  padding: 10px;
}

Exercice libre — Personnaliser son site

30 minutes pour que chaque stagiaire personnalise son site : couleurs, polices, mise en page. L’objectif est que chaque site soit reconnaissable et différent des autres.

Transition vers OpenCode

“Votre site a une structure qui marche. Maintenant on va demander à une IA de l’améliorer — et on va comparer.”

Exercice en deux temps :

  1. Copier le contenu de index.html dans OpenCode avec le prompt :
    “Améliore le style CSS de cette page HTML pour qu’elle soit plus professionnelle et lisible. Garde la même structure.”

  2. Comparer le résultat avec la version manuelle :

    • Qu’est-ce que l’IA a fait différemment ?
    • Y a-t-il des choses que vous n’auriez pas pensé à faire ?
    • Y a-t-il des choses que vous préférez dans votre version ?

Session 3 — GitHub Pages (intégré à l’atelier Cyber)

Créer le dépôt GitHub

  1. Se connecter sur github.com
  2. Cliquer sur New repository
  3. Nom du dépôt : mon-site (ou prenom-stage)
  4. Cocher Public
  5. Ne pas initialiser avec README
  6. Cliquer Create repository

Pousser le site depuis VS Code

Dans le terminal VS Code (Ctrl+ù) :

git init
git add .
git commit -m "Premier commit - mon site de stage"
git branch -M main
git remote add origin https://github.com/TON_PSEUDO/mon-site.git
git push -u origin main

Remplacer TON_PSEUDO par le pseudo GitHub du stagiaire.

Activer GitHub Pages

  1. Sur GitHub, aller dans Settings du dépôt
  2. Section Pages dans le menu gauche
  3. Source : Deploy from a branch
  4. Branch : main / / (root)
  5. Cliquer Save
  6. Attendre 1–2 minutes → l’URL apparaît : https://pseudo.github.io/mon-site

Mettre à jour le site

À chaque nouvel atelier, le stagiaire ajoute une page et la pousse :

git add .
git commit -m "Ajout page Scratch"
git push

GitHub Pages se met à jour automatiquement en 1–2 minutes.

Kit d’animation

Question d’ouverture — Session 1

“Si vous deviez expliquer à quelqu’un ce qu’est une page web, vous diriez quoi ?”

Production attendue

Analogies libres (feuille de papier, affiche, livre…). On part de leurs réponses pour introduire HTML comme “le squelette” — ce qui structure le contenu, sans se préoccuper de l’apparence.

Fallback si blanc

Ouvrir n’importe quelle page web, clic droit → Afficher le code source. Demander : “Vous reconnaissez des mots ?”

Question d’ouverture — Session 2 (CSS)

“Votre site marche, mais il est moche. Comment on pourrait l’améliorer ?”

Production attendue

Liste libre : couleurs, polices, images, mise en page… On en fait le plan de la session CSS.

Fallback si blanc

Montrer deux versions du même site (sans CSS / avec CSS). Demander : “Laquelle vous préférez et pourquoi ?”

Question débrief OpenCode

“L’IA a proposé quelque chose. Est-ce que vous l’avez pris tel quel, modifié, ou rejeté ? Pourquoi ?”

Production attendue

Discussion sur l’IA comme outil, pas comme oracle. On peut la corriger, la guider, choisir de ne pas la suivre.

Matériel détaillé

  • Comptes GitHub créés en amont (ou 15min lundi matin)
  • VS Code installé sur les machines
  • Extension VS Code : Live Server (aperçu instantané dans Chrome)
  • OpenCode installé et configuré
  • Chrome ou Firefox comme navigateur par défaut
