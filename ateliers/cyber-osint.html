Cyber / OSINT — Enquête numérique

Durée : 1 session (3h) + 30min GitHub Pages
Matériel : Ordinateurs, accès internet
Groupes : Groupe A (S1 ven matin), Groupe B (S3 jeu après-midi)

Objectif pédagogique

À la fin de cet atelier, les stagiaires auront mené une enquête OSINT complète sur un personnage fictif en 4 actes, compris ce qu’un inconnu peut trouver sur eux en ligne, et sécurisé leur propre site GitHub Pages.

Ce qu’ils doivent trouver — étape par étape

Acte 1 — WhatsMyName

  • Aller sur whatsmyname.app
  • Chercher le pseudo amensch42
  • Résultat : profil GitHub détecté → ils ont un point d’entrée

Acte 2 — Google Dorks

  • site:github.com "amensch42" → trouve le profil et les repos publics
  • "arthur mensch" développeur → cherche des mentions sur le web
  • "arthur.mensch@proton.me" → cherche si l’email traîne quelque part
  • Résultat : confirmation du profil, repos publics, bio avec la rue

Acte 3 — GitHub

  • Explorer le profil amensch42
  • Lire la bio → trouver la rue
  • Regarder les repos publics → centres d’intérêt, compétences
  • Résultat : adresse partielle + avatar mystérieux

Acte 4 — Street View

  • Aller sur Google Maps → Street View → remonter la rue trouvée dans la bio
  • Chercher un mur avec 4 pattes de chat peintes
  • Résultat : localisation précise de “l’adresse” d’Arthur

Kit d’animation

Question d’ouverture

“Si quelqu’un cherche votre nom sur Google en ce moment, il trouve quoi ?”

Production attendue

Réponses variées : rien / photos / profils réseaux / résultats scolaires… Discussion sur ce qu’on contrôle ou pas.

Fallback si blanc

Faire une recherche Google en direct sur un nom commun fictif. Montrer les types de résultats. Demander : “C’est normal que ça soit accessible ?”

Mission briefing

“Vous êtes enquêteurs. Voici une adresse email : arthur.mensch@proton.me. Retrouvez tout ce que vous pouvez sur cette personne. Vous avez 1h.”

Production attendue

Ils partent en autonomie. Ne pas donner les outils tout de suite — laisser 5 min de recherche libre avant d’introduire WhatsMyName si personne ne trouve.

Fallback si bloqués dès le départ

“Par où commenceriez-vous si vous cherchiez quelqu’un que vous connaissez ?”

Puis introduire WhatsMyName : “Il existe des outils qui cherchent un pseudo sur des centaines de plateformes en même temps.”

Question acte 2 — Google Dorks

“Google c’est juste une barre de recherche ? Ou on peut faire des choses plus précises ?”

Production attendue

Discussion sur les opérateurs : site:, "guillemets", filetype:. On construit ensemble les dorks à utiliser.

Fallback si blanc

Démo en direct : taper site:github.com "amensch42" et montrer la différence avec une recherche simple.

Question acte 4 — Street View

“Vous avez une rue. Comment vous feriez pour trouver exactement où habite Arthur sans y aller ?”

Production attendue

Ils pensent à Maps, Street View, photos satellites… On les laisse trouver la méthode avant de montrer Street View.

Fallback si blanc

Ouvrir Google Maps, chercher la rue, passer en Street View. Demander : “Vous cherchez quoi sur ce mur ?”

Question débrief — après la mise en commun

“Tout ce que vous avez trouvé sur Arthur — est-ce qu’on pourrait en trouver autant sur vous ?”

Production attendue

Prise de conscience personnelle. Discussion sur les traces involontaires : pseudo identique partout, email dans une bio, photo avec métadonnées GPS…

Fallback si blanc

“Est-ce que vous utilisez le même pseudo sur plusieurs plateformes ?”

Question GitHub Pages

“Votre site est en ligne. N’importe qui peut le voir. C’est un problème ?”

Production attendue

Discussion sur ce qu’on veut partager ou pas, la notion de données personnelles, pourquoi passer en privé ou rester public. Lien avec l’enquête : si Arthur avait mis son repo en privé, qu’est-ce qu’on n’aurait pas trouvé ?

Fallback si blanc

Montrer les paramètres de visibilité d’un repo GitHub. Expliquer la différence entre public (indexé par Google) et privé.

Note débrief — le clin d’œil Mistral

En fin d’atelier, révéler que “Arthur Mensch” est aussi le vrai nom du co-fondateur de Mistral AI — la startup française qui a créé les modèles de langage open source. Ouvre une discussion sur l’IA française, l’écosystème tech européen, et le fait que des jeunes français ont créé une des boîtes IA les plus importantes du monde.
