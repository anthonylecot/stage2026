Agenda Ateliers Checklist

Prompt Engineering — Améliorer sa page de présentation

Durée : 1h
Matériel : Ordinateurs, OpenCode installé, site HTML en cours
Groupes : Groupe A (S1 jeu matin), Groupe B (S3 lun matin)

Objectif pédagogique

À la fin de cet atelier, les stagiaires comprennent que la qualité d’un prompt change radicalement le résultat de l’IA — et ont amélioré leur page de présentation HTML.

Timing

Durée Étape
0:00 Question d’ouverture
0:10 Chacun écrit sa bio en 3 lignes (sans IA)
0:20 Premier prompt libre → résultat
0:30 Comparer les prompts entre stagiaires
0:40 Itérations : ajouter du contexte, un ton, une contrainte
0:50 Intégrer le meilleur résultat dans le site HTML
0:55 Tour de table : c’était quoi votre meilleur prompt ?

Kit d’animation

Question d’ouverture

“Vous avez déjà utilisé ChatGPT ou une IA ? Vous lui avez demandé quoi ?”

Production attendue

Expériences variées, bonnes et mauvaises. On fait émerger la notion que “ça dépend comment on demande”.

Fallback si blanc

Faire une démo en direct : même question posée de deux façons différentes. “Écris une bio.” vs “Écris une bio de 3 phrases pour un lycéen de 15 ans qui aime le sport et les jeux vidéo, sur un ton décontracté.” Comparer les résultats.

Question mi-parcours

“Votre prompt a donné un résultat nul. Qu’est-ce que vous changeriez ?”

Production attendue

Ils identifient ce qui manque : contexte / ton / format / longueur / contrainte. On construit ensemble une grille de ce qui fait un bon prompt.

Fallback si blanc

Proposer la grille directement : Contexte + Rôle + Format + Contrainte. Leur demander d’enrichir leur prompt avec ces 4 éléments.

Matériel détaillé

  • OpenCode installé et fonctionnel sur chaque machine
  • Site HTML de chaque stagiaire ouvert et prêt
